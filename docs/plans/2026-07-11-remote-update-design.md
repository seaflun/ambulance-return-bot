# 公務電腦遠端更新設計

日期：2026-07-11

## 目標

在 NAS 的「救護 Worker 後台」新增遠端更新功能，讓管理者可下達更新命令，由公務電腦 Worker 主動接收並在安全時機完成更新。

現有 Worker GUI 的「檢查更新」按鈕完整保留，手動與遠端更新共用同一套既有更新器及版本、SHA256 驗證流程。

## 已確認需求

- 公務電腦當下有案件或自動登打工作時，勤務永遠優先。
- 所有工作結束後，遠端更新才可開始。
- 自動更新必須在背景執行，不顯示黑色命令視窗、不跳出提示，也不搶走目前視窗焦點。
- 為避免使用者正在填寫尚未送出的資料，Worker 還要確認 Windows 已無鍵盤或滑鼠操作一段時間。
- 手動「檢查更新」按鈕及其操作方式保持不變。

## 方案比較

### 方案 A：Worker 主動輪詢 NAS 命令（採用）

NAS 儲存一筆有編號的更新命令，Worker 在現有每 10 秒輪詢中一併檢查。符合安全條件後，Worker 以隱藏程序呼叫既有更新器。

優點：不需開放公務電腦連入埠、不需 WinRM 或遠端桌面、可沿用 Worker Token、記憶體與 CPU 增量極小，也能可靠追蹤狀態。

### 方案 B：NAS 主動連入公務電腦

使用 WinRM、PowerShell Remoting 或排程工作從 NAS 推送更新。

缺點：需要開放公務電腦連入服務與管理權限，網路與資安設定較複雜，故不採用。

### 方案 C：固定時間自動檢查更新

Worker 每天固定時段自行更新，不由 NAS 下達命令。

缺點：管理者無法控制更新時機，也無法清楚確認某次遠端要求是否完成，故不採用。

## 操作流程

1. 管理者在 NAS「救護 Worker 後台」按下「遠端更新公務電腦」。
2. NAS 建立唯一命令編號並顯示「等待公務電腦接收」。重複按鈕不會建立第二筆進行中的命令。
3. Worker 使用現有 `X-Worker-Token` 取得命令，並回報目前套件版本與 Worker ID。
4. Worker 逐項確認安全條件：
   - 沒有正在執行的自動領取案件。
   - `MANUAL_TASK_ACTIVE` 未啟用。
   - `manual_task_active.lock` 未啟用。
   - 沒有本機案件查詢或 Selenium 登打工作。
   - Windows 鍵盤與滑鼠已閒置至少 120 秒。
5. 若任一條件不符合，命令保留並顯示「等待勤務完成」或「等待電腦閒置」，Worker 每 10 秒重新判斷。
6. 條件符合後，Worker 先回報「開始更新」，再用完全隱藏的背景程序啟動更新。
7. 更新器沿用既有 GitHub Release 版本與 SHA256 驗證、備份、覆蓋及重啟流程。
8. Worker 重啟後讀取更新結果，向 NAS 回報「更新完成」或「更新失敗」及目前版本。

Windows 閒置秒數以 `REMOTE_UPDATE_IDLE_SECONDS` 設定，預設為 120 秒；必要時可在環境設定調整。

## NAS 後台顯示

後台新增獨立的遠端更新卡片，包含：

- 目前公務電腦版本。
- 遠端更新按鈕。
- 命令編號與下達時間。
- Worker ID 與最後回報時間。
- 狀態：等待接收、等待勤務完成、等待電腦閒置、更新中、完成、失敗或逾時。
- 更新前版本、更新後版本與錯誤說明。

後台只送出命令，不會直接連線或控制公務電腦。

## 背景更新與使用者體驗

遠端更新不使用現有批次檔視窗，而是以 `CREATE_NO_WINDOW` 與 PowerShell 隱藏模式啟動一個遠端更新包裝器。包裝器再呼叫既有 `update_package.ps1`，因此不會出現黑色視窗或將焦點切離使用者正在操作的程式。

現有手動按鈕仍呼叫 `UPDATE_PACKAGE.bat`，不改變使用者原本的操作習慣。

更新真正開始時會停止並重啟 Worker GUI 與本機網頁，因此必須同時通過「無勤務」與「Windows 已閒置」兩層判斷。

## 狀態與容錯

- NAS 使用原子寫入保存單一進行中命令及最近結果，避免容器重啟後遺失。
- 每筆命令使用 UUID；Worker 對同一命令最多啟動一次更新。
- Worker 在本機 `%LOCALAPPDATA%\AmbulanceReturnBot` 保存更新結果，不受套件覆蓋影響。
- 若下載、SHA256、解壓縮或覆蓋失敗，包裝器保存錯誤與退出碼；Worker 下次輪詢時回報 NAS。
- 若 Worker 離線，命令維持等待狀態，不會造成公務電腦端額外動作。
- 只有通過 Worker Token 驗證的 Worker 能取得或回報更新命令。

## 測試與驗證

- NAS 後台可以建立命令，且不會重複建立進行中命令。
- 未帶正確 Worker Token 時，命令取得與回報端點回傳 403。
- 有勤務、手動工作鎖、本機案件查詢或近期使用者輸入時，不會啟動更新。
- 所有安全條件符合時，只啟動一次隱藏更新程序。
- 更新成功、已是最新版與更新失敗都能在 NAS 後台看到最終結果。
- GUI 原有「檢查更新」按鈕仍能找到並啟動 `UPDATE_PACKAGE.bat`。
- 完成後執行完整測試、建立公務電腦更新包、發布 GitHub Release，並依既有流程驗證下載檔 SHA256、套件內容、NAS 部署雜湊與 `/status`。
